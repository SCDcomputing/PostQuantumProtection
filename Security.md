# Security Policy

## Supported Versions

We actively monitor and secure the following versions of this project:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a potential security vulnerability within this project, please do **not** open a public issue. Instead, report it responsibly to help ensure the security of the project.

### How to Report

Please submit a detailed report containing:
- A description of the vulnerability and its potential impact.
- Step-by-step instructions (or a proof-of-concept script) to reproduce the issue.
- Any potential mitigations or fixes you suggest.

You can send your security report via email to: **[Your Security Email Address]**

### Our Commitment

We take all security reports seriously. After receiving a report, you can expect:
- An acknowledgment of receipt within 48 hours.
- A status update outlining the next steps and estimated timeline for a fix.
- An opportunity to review the fix before it is publicly disclosed.
